# L1 creative tests — Round 2 (planned / running)

*Built off the early L1 learnings. Each test changes **one thing** so the result is attributable. Shareable doc (with the creatives embedded): `L1_Next_Tests.docx`; companion deck: https://shivakimothi-design.github.io/wiom-l1-creative-learnings/*

## Test 1 — Does the "recharge-device" land the category? *(offline / comprehension)*
- **Who:** L1 ready buyer.
- **Hypothesis:** a router + WiFi-signal + recharge-day-cards device reads as "recharge-wala ghar ka net" on its own — and adding a family makes the category land better.
- **The one variable:** family present or not (device-only vs device + family); device element held constant.
- **Objective:** offline comprehension panel (~15–20 per version), not in-market.
- **Success metric:** category-ID — % unaided "ghar ka net on recharge" (and **not** "mobile recharge"); does the family lift it.
- **Decision rule:** lands clean → reuse this device element across statics; family lifts → always include a family. **Auto-kill:** reads as "mobile recharge."
- **Creatives:** A device-only · B device + family.

## Test 2 — इशारों winner: combo, the line, or the graphic? *(in-market — Learning + Scale)*
- **Who:** L1.
- **Hypothesis:** the इशारों-पे winner works as a **combo** of the control line + the WiFi-symbol device, not either alone.
- **Arms (one element changes each):**
  - **A — control:** original "घर का नेट अब आपके इशारों पे" + WiFi-symbol-with-day-chips (live winner, T-043).
  - **B — same line, swap graphic:** "इशारों पे" + **calendar** device.
  - **C — same graphic, swap line:** WiFi-symbol device + "**अपने हिसाब से रिचार्ज करो**" (control framing).
  - **D — same graphic, swap line:** WiFi-symbol device + "**घर का नेट अब रिचार्ज पर / जितने दिन चाहिए**" (plain explanation).
- **Held constant:** photo, layout, lockup; exactly one element per arm.
- **Objective:** in-market, run in **both** Learning (clean attention) and Scale (bookings vs the original).
- **Success metric:** Learning = CTR + install-rate; Scale = CPBL vs the original T-043 and the pool median (≥5 bookings).
- **Decision rule:** only A wins → it's the **combo**. B ≈ A → graphic is swappable (line carries it). C/D ≈ A → line is swappable (graphic carries it). **Auto-kill:** misread.

## Test 3 — Does the carousel work on scale? *(scale)*
- **Who:** L1.
- **Hypothesis:** a good carousel can produce bookings on scale — not just a Learning-phase hook.
- **The one variable:** carousel format on the Scale campaign (card-2 resolution improved). *Note: copy improved too → directional "can a good carousel scale?" probe, not a clean carousel-vs-static read.*
- **Held constant:** keep card-2 (the "ghar ka net + recharge calendar" resolution) constant, so later card-1 hook iterations are attributable.
- **Objective:** Scale (BFC-VOLUME).
- **Success metric:** CPBL vs the Delhi pool median (≥5 bookings) + first-recharge + watch comments for the "mobile recharge" misread.
- **Decision rule:** clears the median with healthy recharge → carousel works on scale → then iterate card-1 hooks (holding card-2 constant); if it lags → stay with statics. **Auto-kill:** misread.
- **Creative:** WiFi-नहीं carousel (T-046) — card-1 curiosity hook → card-2 RWGKN explain.
