# L1 statics wave — early learnings + next steps

- **Wave:** June L1 statics (BRIEF-UNFLT-20260603) · 9 concepts (C-038/039/040, T-041–T-046)
- **Status:** ⚠️ **EARLY — ~3–4 days of data.** Signals, not verdicts. Bookings are still small and first-recharge data isn't in yet. **This will be re-read as the data grows over the next ~2 days**, and the ranking may change.
- **Read on:** Learning campaign = creative pull (CTR / install rate) · Scale campaign = cost-per-booking vs the pool median (Delhi-clean).

## Learnings

1. **Anchor — Hindi "घर का नेट" pulls cheaper than English "Home net".** 🔶
   Same router ad, one word changed. Hook tied (CTR 0.50 vs 0.49), but "Ghar ka net" installs at **₹18 vs ₹24**. → Lean "Ghar ka net". *(T-041 vs T-042)*

2. **Format must fit the idea — it flips by theme.** 🔶
   A simple scene → **static** wins (0.48% vs 0.23% as a carousel). A "WiFi नहीं" claim → **carousel** wins (0.48% vs 0.36% as a static). No format is universally better. *(T-043/T-044, T-045/T-046)*

3. **A contrast / comparison hook stops the scroll; a plain scene doesn't.** 🔶
   "WiFi नहीं" contrast (0.48%) and the need-toggle split-screen (0.65%) both pull hard vs a plain scene (0.23%). Echoes pre-reset, where comparison brought the contrast that drove attention. → Build more contrast/comparison. *(C-038, T-045)*

4. **Open question — in "WiFi नहीं", is it the element or the hook?** ❓
   The crossed-out-WiFi *element* (+ 1/2/7-din options) and the curiosity *words* always move together, so we can't yet attribute the attention. → Isolation test (below).

5. **Clarity wins bookings; curiosity wins attention.** 🔶
   The clean इशारों-पे static, which **shows the recharge day-options on the image**, books cheapest of the 27 live ads (**₹1,366, #1**). The flip side: "WiFi नहीं" as a **lone static** is the pool's **worst at booking (₹9,602, #23)** — it stops the scroll but one frame can't resolve the destination (mobile-recharge misread bites). The same idea **as a carousel** converts well. *(T-043 vs T-045)*

6. **The pre-launch scorecard isn't predictive yet.** 🔶
   The two highest-scored creatives (T-045 8.0, T-046 8.2) included the pool's *worst* scale performer (T-045). Watch whether the rubric over-rewards craft and under-prices misread risk. *(Validate across ~30+ creatives — see the evaluation framework.)*

**Headline:** *curiosity wins attention, clarity wins bookings — and the "WiFi नहीं" idea only converts in carousel form, never as a lone static.*

## Next steps

### Do now (clear from the data, no test needed)
1. Write **"घर का नेट"** (Hindi), not "Home net", on every ad.
2. Run **"ये WiFi नहीं"** only as a 2-card carousel, never a lone image.
3. Show the **recharge day-options on the image itself**, every time.

### Tests to run (each changes one thing)
- **Test 1 — Why does "WiFi नहीं" grab attention?** (run first)
  A: crossed-WiFi *picture* + plain words · B: curious *words* + no crossed picture. → whichever wins on clicks is what to reuse.
- **Test 2 — Why does इशारों-पे book cheapest?** (with Test 1)
  A: इशारों-पे line + WiFi-symbol element (as-is) · B: same line + calendar element · C: same element + plain line. → A vs B = does the element matter? A vs C = does the "control" message matter?
- **Test 3 — More curiosity carousels.** 3–4 carousels, different curious card-1, same card-2 (घर का नेट · 1/7/14 din). → which hook pulls best.
- **Test 4 — More comparison ads.** 2–3 more split-screen "this vs that" ads. → does comparison keep winning attention.

### Before scaling spend on any winner
1. **Week 2–3:** do the cheap bookings actually **recharge**? (A booking that never recharges is a fake win.)
2. **Misread check:** ask a few people "what is this ad for?" — nobody should say "mobile recharge."
